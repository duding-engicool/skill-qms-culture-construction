# 质量文化建设辅助技能

## 一、能力边界与调用前置说明（通用规则 1 & 5）

### 本技能能做什么
- 质量文化成熟度**诊断**：基于 10 个文化维度自评，输出成熟度等级（1-5 级）与短板画像
- 落地**路线图**设计：按成熟度匹配破冰 / 机制固化 / 文化内化三阶段 12 个月实操活动
- **活动策划**：质量承诺墙、质量之星、积分制、红黑榜、改善提案等可落地模板
- **行为考核清单**：可直接套用的质量行为加分/减分体系（考核"行为"而非"结果"）
- **案例参考**：5 个真实行业案例的可模仿关键动作
- **战略对齐校验**：若用户提供质量方针/战略，校验文化举措是否服务于战略质量缺口

### 本技能不能做什么（边界）
- ❌ 不能代替真实的员工调研/访谈（自评仅作初判，正式诊断建议顾问访谈）
- ❌ 不能代替 HR 制度落地与绩效系统实施
- ❌ 不能保证文化改变效果（文化养成周期以年计，受组织多重因素影响）
- ❌ 不能出具认证/审核结论（本技能是建设辅导，非符合性审核）
- ❌ 不能脱离企业实际照搬活动（所有活动须按行业/规模裁剪）

### 开始使用前需要的前置信息
- **必填（影响诊断准确度）**：企业所属行业、大致规模（人数）、当前质量管理痛点描述
- **选填（影响路线图战略契合度）**：已发布的质量方针文本、企业质量战略方向
- 上述信息若缺失，技能会用行业常见水平做假设，并**显式标注「供参考 / 待确认」**；但行业与规模缺失会显著降低诊断针对性（见下方"失真提示"）

> ⚠️ **失真提示（关键）**：本技能的诊断问卷与路线图模板基于质量管理通用规律。
> 若**未提供企业质量方针/战略**，输出的路线图是按"通用最佳实践"生成的，
> 可能与企业真实的战略质量缺口存在**偏差（失真）**——即"该做的都列了，但未必是企业当下最该先做的"。
> **强烈建议**在启动路线图前补充质量方针，技能会在每阶段活动后标注「待提供方针后复核」，
> 待方针到位后重新校验优先级。方针辅助见 `qms-policy-assistant` 技能。

## 二、输入载体判断（通用规则 2）

- **交互式问答（默认）**：用户口头描述行业/规模/痛点，技能引导完成 10 题自评
- **文档上传（可选）**：若用户提供已有的《质量文化调研表 / 员工意识问卷 / 现有文化建设方案》（`.docx` / `.xlsx` / `.md` / `.txt`）
  - `.docx` → 调用 **docx** 技能解析为文本
  - `.xlsx` → 调用 **markitdown** 技能或读取表格，抽取已有评分/结论
  - 解析后由技能比对现有结论与本次诊断，输出**差异提示**（不做凭空重写）

## 三、缺失信息处理（通用规则 3）

- 任何以假设补足的信息，输出时一律带标记：
  - `【供参考】`：基于行业常见水平的推测值，需用户确认
  - `【待确认】`：影响结论但暂未获得的关键信息
  - `【待提供方针后复核】`：战略对齐相关结论，待方针文本到位后重新校验
- 若用户执意不提供某项必要信息，该部分在最终产物中明确标注上述标记，不伪装为确定结论。

## 四、输出形式与大纲确认（通用规则 4）

- **默认输出**：对话内直接给诊断结果 + 推荐阶段 + 关键活动摘要（文字版，无需确认即可给）
- **文档输出（可选）**：若用户需要《质量文化建设整体方案》（.docx / .md）
  - **强制先给大纲**：生成前必须展示文档大纲结构，用户确认后再生成全文
  - 大纲骨架见 `assets/文化建设方案_输出模板.md`

## 五、工作流

### 第一步：收集企业信息
引导用户提供：行业、规模、当前质量管理状态、最大痛点。信息不足时按行业假设并标注「供参考/待确认」。

### 第二步：执行成熟度诊断
请用户就以下 10 个维度按 0-5 分自评（5=完全符合，0=完全不符合）：

| # | 文化维度 | 对应条款要点 | 0分 vs 5分 |
|---|----------|--------------|------------|
| 1 | 问题响应思维 | 5.1 领导作用（解决问题导向） | 追责 vs 解决 |
| 2 | 主动报告意愿 | 7.3 意识 / 10.2 改进 | 隐瞒 vs 主动报 |
| 3 | 质量目标分解 | 6.2 目标 / 7.3 意识 | 仅部门级 vs 到岗 |
| 4 | 新人质量培训 | 7.2 能力 / 7.3 意识 | 只教操作 vs 有意识教育 |
| 5 | 领导质量重视度 | 5.1 领导作用 | 交期第一 vs 质量优先 |
| 6 | 跨部门协同 | 5.1 / 7.4 沟通 | 推诿 vs 协同 |
| 7 | 质量复盘机制 | 10.2 改进 / 9.1 监视 | 开完就忘 vs 闭环 |
| 8 | 改善建议氛围 | 7.3 意识 / 10.3 改进 | 打击 vs 鼓励 |
| 9 | 根因分析深度 | 10.2 改进 | 找替罪羊 vs 真根因 |
| 10 | 质量文化载体 | 7.3 意识（象征物） | 无 vs 有载体 |

调用诊断脚本（详见 `scripts/diagnose.py`）：
```bash
python scripts/diagnose.py --scores "3,2,4,1,2,3,2,1,2,3"
# 或读取已填 JSON：
python scripts/diagnose.py --input '{"scores":[3,2,4,1,2,3,2,1,2,3],"industry":"制造业"}'
```

### 第三步：输出诊断结果与路线图
- 成熟度五级模型与三阶段路线图（破冰 / 机制固化 / 文化内化）详见 `references/roadmap_playbook.md`
- 按诊断等级推荐启动阶段：L1-2→第一阶段；L3→第二阶段；L4-5→第三阶段

### 第四步：输出行为考核机制
详见 `references/assessment.md`（考核"行为"而非"结果"，考核框架、加分/减分清单、实施指南）

### 第五步：案例参考
详见 `references/cases.md`（5 个真实行业案例，含可模仿关键动作）

## 六、战略对齐校验（可选，但强烈建议）

- **用户提供了质量方针/战略** → 校验每阶段活动是否服务于方针中的战略质量锚点；无关活动标注「战略弱相关，建议后置」
- **用户未提供** → 路线图照常生成，但每阶段末尾标注 `【待提供方针后复核】`，并弹出本节开头的"失真提示"
- 与 `qms-policy-assistant`、`qms-objective-decomposition` 形成「方针 → 目标 → 文化」递进链路

## 七、资源索引
- 诊断脚本：`scripts/diagnose.py`（10题评分→等级判定+短板推荐）
- 文化条款映射：`references/iso9001_clauses.md`（7.3/5.1/7.2/7.4/10.3）
- 路线图手册：`references/roadmap_playbook.md`（五级模型+三阶段活动表）
- 行为考核清单：`references/assessment.md`
- 案例库：`references/cases.md`
- 方案输出模板：`assets/文化建设方案_输出模板.md`

## 八、注意事项
- 诊断问卷可由用户自评，也可由顾问访谈后填写，推荐后者更准确
- 路线图可根据企业实际加速或延长，但验证节点不可跳过
- 考核机制落地前需与 HR/管理者充分沟通，避免执行阻力
- 案例仅供参考，企业情况不同效果会有差异
